# Chebbi Trading — Design Context

## Users

**Who**: Retail forex and cryptocurrency traders across North Africa, the Middle East, and francophone Europe. Ages ~20–45, mostly mobile-first, varying from beginners to intermediate traders. Many discovered the platform through YouTube live streams or Telegram.

**Context**: Users arrive seeking free, verifiable trading signals and community access. They evaluate credibility through documented performance data (yearly/monthly results, win rates, pips), social proof (testimonial stars, member counts), and live YouTube presence. They register by opening an XM brokerage account through an affiliate link, then submit their XM ID for verification.

**Job to be done**: Quickly assess the platform's legitimacy, register for free signal access, browse documented trading results, and stay informed through the blog and crypto VIP sections.

**Emotional goals**: Build **confidence** in the platform's track record, feel **exclusivity** when joining the community, experience **trust** through transparency of live-documented results, and feel the **urgency** to act via the registration CTA flow.

---

## Brand Personality

**Three words**: Professional · Transparent · Authoritative

**Voice & Tone**:
- Direct, numbers-driven communication — always lead with data.
- Confident but not arrogant; the results speak for themselves.
- Warm community feel balanced with financial seriousness.
- The founder (Amine Chebbi) is the face — personal credibility matters.

**Languages**: French (primary), English, Arabic (RTL). All content must be trilingual. French is the default fallback.

## Aesthetic Direction

### Theme
**Dark-mode dominant**. The default and intended experience is dark (`#06090f` background, `#111827` cards). Light mode exists as a toggle but is secondary. All design decisions should be optimized for dark backgrounds first.

### Color System

| Token | Hex | Purpose |
|-------|-----|---------|
| `--primary` (emerald) | `#10b981` | Success, growth, profit, primary accents, active states |
| `--ct-green-light` | `#34d399` | Gradient endpoints, lighter accents |
| `--ct-xm` (XM Red) | `#d42127` | XM broker branding, registration CTAs, Step 1 |
| `--ct-red` | `#ef4444` | Destructive actions, losses, errors |
| `--ct-blue` | `#3b82f6` | Email links, informational accents |
| `--ct-gold` | `#f59e0b` | Badges, highlights, special emphasis |
| `--ct-purple` | `#8b5cf6` | Charts (chart-4), decorative variety |
| `amber-400/500` | `#fbbf24`/`#f59e0b` | Crypto VIP section — entirely amber-themed, distinct from forex green |
| `--muted-foreground` | `#94a3b8` (dark) | Body text, descriptions, secondary content |
| `--foreground` | `#e2e8f0` (dark) | Headings, primary text |
| `--background` | `#06090f` (dark) | Page background — near-black with blue undertone |
| `--card` | `#111827` (dark) | Card surfaces, elevated containers |
| `--border` | `rgba(255,255,255,0.07)` (dark) | Subtle glass-like borders |

**Key rule**: Forex sections use **emerald green** as primary. Crypto VIP section uses **amber/gold** as primary. These are two distinct visual identities within the same platform.

### Typography
- **Font family**: `Inter` (loaded via Google Fonts), fallback to system `ui-sans-serif`.
- **Mono**: `JetBrains Mono` for numerical/financial data (percentages, pips, prices).
- **Heading weight**: `font-extrabold` (800) and `font-black` (900) — headings are always heavy.
- **Body**: `text-sm` (14px) default, `leading-relaxed` for readability.
- **Sizing scale**: Hero `text-4xl` → `text-6xl`, Section headings `text-3xl` → `text-5xl`, Card titles `text-base` → `text-lg`.
- **Labels**: Uppercase, `text-[10px]`, `tracking-wider`, `font-bold`, `text-slate-300` — used for form labels and metadata.

### Spacing & Layout
- **Max width**: `max-w-7xl` (1280px) for main content, `max-w-4xl` for focused content (crypto, FAQ).
- **Section padding**: `py-16 lg:py-20` standard, `py-16 lg:py-24` for emphasis.
- **Horizontal padding**: `px-5` universally.
- **Card padding**: `p-6` standard, `p-6 md:p-7` or `p-6 md:p-8` for emphasis.
- **Grid gaps**: `gap-4 lg:gap-6` for dense grids, `gap-10 lg:gap-16` for section-level grids.
- **Border radius**: `rounded-xl` (12px) for buttons and inputs, `rounded-2xl` (16px) for cards, `rounded-full` for avatars and pills.

### Visual Effects
- **Glassmorphism**: `backdrop-blur-[20px]` (`.glass` utility) for navbar on scroll.
- **Radial glows**: Large `blur-[120px]` divs with `bg-primary/5` or `bg-ct-xm/5` for ambient background lighting.
- **Grid pattern**: Subtle `opacity-[0.03]` grid overlay on hero sections.
- **Colored shadows**: `shadow-lg shadow-primary/10`, `shadow-xl shadow-ct-xm/40` for interactive elements.
- **Green gradient text**: `.text-gradient-green` — `linear-gradient(135deg, #10b981, #34d399)` with `bg-clip-text text-transparent`.
- **Amber gradient text** (crypto): `bg-gradient-to-r from-amber-400 to-orange-400 bg-clip-text text-transparent`.
- **Subtle top borders**: `h-1 bg-gradient-to-r from-primary via-primary to-ct-green-light` on cards.

### Animation Patterns
- **Framework**: Framer Motion exclusively. No CSS-only animations except for keyframe utilities (preloader, ticker, float).
- **Scroll reveals**: `SectionReveal` / `FadeIn` components using `useInView` with `{ once: true, margin: '-80px' }`.
- **Reveal motion**: `{ opacity: 0, y: 30 } → { opacity: 1, y: 0 }` with `duration: 0.6, ease: 'easeOut'`.
- **Stagger pattern**: `staggerChildren: 0.1, delayChildren: 0.1` for hero content groups.
- **Accordion**: `AnimatePresence` with `height: 0 → auto` and `opacity: 0 → 1`, `duration: 0.3`.
- **Hover interactions**: `-translate-y-0.5` lift on CTAs, `hover:shadow-lg` expansion, `hover:border-primary` highlight on controls.
- **Active nav indicator**: `layoutId="activeNav"` for animated underline transitions.
- **Floating elements**: CSS `@keyframes floatBob` (4s ease-in-out infinite translateY) for Telegram button.
- **Preloader**: Pulsing logo with scale(1→1.08), sliding progress bar, fade-in-out text.

### Interactive Elements
- **Primary CTA** (XM Register): `bg-ct-xm text-white`, bold red with `shadow-ct-xm/40` glow on hover. Always includes directional icon (ArrowRight).
- **Secondary CTA** (YouTube): `bg-background border border-border`, ghost style with `hover:border-primary` transition.
- **Form inputs**: `bg-[#0b1018] border-slate-800/60 focus:border-primary/50 rounded-xl px-4 py-5` — ultra-dark inputs against dark cards.
- **Buttons**: `rounded-xl` (not `rounded-lg`), `font-bold`, always with icon + text.
- **Badges**: `bg-primary/10 text-primary border-primary/20 px-4 py-1.5 text-sm font-semibold` — subtle pill style.
- **Status indicators**: Green `animate-pulse` dots for "live" status. Pinging circles (`animate-ping`) for emphasis.

### Iconography
- **Library**: Lucide React exclusively. Size `14` for inline/compact, `16` for standard, `18–20` for emphasis.
- **Emoji usage**: Extensive for section badges, stat cards, and highlight chips. Adds personality without requiring image assets.

## Component Patterns

### Cards
```
bg-card border border-border rounded-2xl
hover:-translate-y-1 hover:shadow-xl hover:shadow-primary/5 transition-all duration-300
```

### Section Structure
```
<section className="py-16 lg:py-20">
  <div className="max-w-7xl mx-auto px-5">
    <SectionHeading badge="..." title="..." subtitle="..." />
    <SectionReveal>
      {/* content */}
    </SectionReveal>
  </div>
</section>
```

### FAQ Accordion Items
Standalone `FaqAccordionItem` component with:
- Icon-badge left + question text + Plus/rotate toggle right
- `AnimatePresence` expansion for answers
- `faq-inner-html` CSS class for styled `dangerouslySetInnerHTML` content
- RTL-aware highlight blocks (`border-left` ↔ `border-right`)

### Trilingual Data
- Static UI: `t('key', language)` from `src/lib/i18n.ts` dictionary
- Dynamic DB content: `pickLang(entity, 'field', language)` which resolves `fieldFr`/`fieldEn`/`fieldAr` suffixes
- RTL: `document.documentElement.dir = language === 'ar' ? 'rtl' : 'ltr'` set in Navbar

## Design Principles

1. **Data Proves Everything**: Performance numbers, win rates, yearly results, and member counts are the hero content. They must be prominent, beautifully formatted (mono font, gradient text), and always accessible within 1-2 scrolls.

2. **Dark Terminal Aesthetic**: The platform should feel like a premium trading terminal. Deep blacks, emerald profit-green, subtle glass effects, and ambient radial glows create a professional, focused atmosphere. Avoid bright whites or playful colors.

3. **Motion with Purpose**: Every animation serves UX — scroll reveals build narrative, accordion expansions show/hide detail, hover lifts signal interactivity. No gratuitous motion. Use `once: true` to prevent re-triggering.

4. **Trilingual by Default**: Every piece of text — UI labels, dynamic content, error messages, and tooltips — must exist in French, English, and Arabic. Arabic layouts must be seamlessly RTL. Never hardcode a string without its trilingual counterpart.

5. **Two Visual Identities, One Platform**: Forex sections are **emerald green**. Crypto VIP is **amber/gold**. These color worlds must not bleed into each other. Each section should feel like its own branded experience while sharing the same dark foundation and component patterns.

6. **Frictionless Conversion**: The registration flow (XM link → form → verification) must be visually guided with numbered steps, colored progress indicators, and immediate feedback (success/error states with icons). The XM red CTA button is the single most important interactive element on the platform.
